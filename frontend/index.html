Current state vs digital-first

Today

  • Team maintains a Word doc + Excel file
  • Published to website as static downloads
  • Users search manually within files
  • Version history is filename-based (v1.1_0.docx)
  • Provider portals re-key data from the Excel
  • Every update requires re-publishing all files
  • No way to embed pricing in other systems without manual effort

Digital-first

  • Structured data (JSON/YAML/Markdown) is the master copy
  • Web interface, API, and static files all generated from the same source
  • Real-time search and filter for all users
  • Version history in Git — full audit trail, reviewable diffs
  • Provider portals query the API directly
  • Publish once, serve everywhere
  • Machine-readable from day one — no re-keying required
